# Task spec #10 — e2e/: end-to-end test suite over the live stack

## Goal
Чёрноящичные E2E-тесты: поднять собранный `apps/api`, прогнать сквозной поток —
signup → коннектор шлёт события (ingest) → профиль строится → read-API отдаёт профиль →
email-кампания (consent-гейт). Опционально гнать UI `apps/console` через Playwright.

## Контекст
Монорепо pnpm (Node 20). `apps/api` запускается `node apps/api/dist/server.js` (env `PORT`,
без `DATABASE_URL` — in-memory сторы, чего достаточно для E2E). Контракт `/v1`: signup, modules,
tenants/:id/modules/:key, track, tenants/:id/{profiles,events}, tenants/:id/email/campaigns, health.

## Стек
TS, Playwright (для UI) и/или прямой HTTP (`fetch`) для API-уровня. Пакет `@cdp-us/e2e`.
Тесты сами спавнят API-процесс на свободном порту и тушат его (setup/teardown). Без внешней сети.

## Сценарии (минимум)
1. signup → получает apiToken + writeKey.
2. track identify+track по writeKey → 200, stored.
3. GET profiles (Bearer) → профиль построен (firmographics из traits).
4. modules enable email (Bearer) → 200.
5. consent marketing_email + email/campaigns → sent для consented, skipped для нет.
6. health → ok.

## Allowed files
- ТОЛЬКО `e2e/**` (package `@cdp-us/e2e`).

## Do-not-touch
- Исходники любых apps/packages/modules (читать поведение можно, ПРАВИТЬ нельзя),
  **`.github/**` (CI-интеграцию e2e добавит API-агент отдельно)**, root `tsconfig.json`, РФ-контент.

## Acceptance
- `pnpm --filter @cdp-us/e2e test:e2e` сам поднимает API, прогоняет сценарии зелёно, тушит процесс.
- Детерминированно, без внешней сети; чистый teardown (не висит порт/процесс).

## Test command
`pnpm install && pnpm build && pnpm --filter @cdp-us/e2e test:e2e`

## Risk
Корректно спавнить/тушить API (free port, kill в teardown). Не редактировать чужой код и `.github`.
DATABASE_URL не задавать (in-memory). Без сети.
