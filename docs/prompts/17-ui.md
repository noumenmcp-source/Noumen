# Task spec #17 — packages/ui: shared React design-system library

## Goal
Общая библиотека React-компонентов для console/admin/marketing (единый стиль). ТОЛЬКО библиотека —
существующие приложения НЕ рефакторить сейчас (адаптация позже, по желанию).

## Стек
TS strict, ESM, React (peerDependency >=18), Tailwind-совместимые классы (или CSS-модули), English,
доступность (a11y). Тесты: React Testing Library + jsdom. Storybook — опционально.

## Компоненты (минимум)
Button, Input, Select, Table, Card, Badge, Alert/Toast, EmptyState, Spinner, AppShell/Nav.
Все: типизированные props, `readonly` где уместно, aria-роли/лейблы, forwardRef где нужно,
tree-shakeable именованные экспорты из `src/index.ts`.

## Allowed files
- ТОЛЬКО `packages/ui/**` (package `@cdp-us/ui`).

## Do-not-touch
- `apps/**` (НЕ рефакторить console/admin/marketing сейчас), `packages/sdk-react/**` и прочие,
  root `tsconfig.json`, `.github/**`, РФ-контент.

## Acceptance
- `pnpm --filter @cdp-us/ui build && test` зелёные (RTL, offline).
- Каждый компонент рендерится и доступен (корректные роли/лейблы); Button onClick, Input onChange,
  Table рендерит строки, EmptyState показывает сообщение — покрыто тестами.
- TS strict, zero `any`; именованные экспорты; React как peerDep (не bundle'ить React).

## Test command
`pnpm install && pnpm --filter @cdp-us/ui build && pnpm --filter @cdp-us/ui test`

## Risk
Не рефакторить существующие apps. React — peerDep. a11y обязательна. Без тяжёлых UI-зависимостей.
