# Промт: исследование функций ECO SAS (feature/market validation)

Готовый к запуску промт для проверки **функционала** ECO SAS рынком: спрос по
каждой функции, покрытие конкурентами, готовность платить, приоритет (MVP vs
later), упаковка по планам. Вставляй целиком в Флот / deep-research. Язык вывода —
русский, рынок — США.

---

## РОЛЬ

Ты — старший product/market researcher по B2B SaaS marketing analytics (США).
Тебе дан полный список функций продукта ECO SAS. Задача — **не пересказать их**, а
проверить рынком: какие функции реально нужны, кому, кто их уже закрывает, за что
готовы платить, что выкинуть. Правила:

- Каждое утверждение — с источником и пометкой `[факт]` / `[оценка]`.
- Приоритет данных 2024–2026, рынок США.
- Не подыгрывай продукту. Если функция — «никому не нужная фича» или давно решена
  конкурентами лучше нас, скажи прямо.
- Длина ≠ доказательство. Одна цифра/цитата с источником ценнее абзаца.

## ПРОДУКТ (одной фразой)

ECO SAS — «переводчик маркетингового ROI» для SMB/mid-market и агентств в США:
агрегирует рекламу/аналитику/выручку/SEO и **объясняет простым языком**, какой
канал приносит деньги, где ломаются CAC/ROAS/payback, что с SEO и конкурентами,
что делать и какой пакет услуг докупить. Ключевое: AI **объясняет посчитанные
метрики** (с цитатами), а не выдумывает советы.

## ФУНКЦИИ ПРОДУКТА (объект исследования)

### A. Подключение данных
- Коннекторы: Google Ads, Meta Ads, GA4, Stripe, Shopify, HubSpot, Search
  Console, DataForSEO, CSV/Google Sheets.
- Авто-синк, контроль свежести данных, **Data Health** (что сломалось/устарело).

### B. Единый профиль клиента
- 3D-профиль клиента (склейка identity между источниками).
- Сегменты клиентской базы.

### C. Channel ROI / прибыльность
- CAC, ROAS, CPA, payback, MER, прибыльность канала.
- Прибыльность на уровне кампании.
- Spend vs revenue по каналам.
- Атрибуция (UTM/click-id, first/last touch).

### D. SEO-интеллект
- Трекинг видимости по ключам (динамика по месяцам/году).
- Branded / non-branded разделение.
- Видимость конкурентов / share of voice.
- Top gained / lost keywords.
- Движение конкурентов («competitor X-ray»).
- SEO opportunity scoring, маппинг на landing-страницы.

### E. AI-объяснения (главный дифференциатор)
- Глоссарий метрик простым языком («что такое органический трафик»).
- Объяснение аномалий («почему ROAS просел в этом месяце»).
- Месячный нарратив по перформансу.
- Объяснение Channel ROI / SEO / движения конкурентов.
- «Что проверить дальше» + рекомендованное действие.
- Founder-friendly режим (язык владельца, не аналитика).

### F. Рекомендации (rule-based)
- Бюджет-реаллокация, creative fatigue, landing/tracking проблемы, SEO-audit
  триггеры — с привязкой к метрикам.

### G. Upsell / пакеты услуг
- Система вскрывает потребность и предлагает пакет: tracking cleanup, SEO audit,
  landing optimization, paid audit, creative testing, attribution setup, content
  cluster, data hygiene, agency strategy call.

### H. Отчёты
- Месячный agency-ready PDF.
- Scheduled email-отчёты.
- White-label брендинг.

### I. Agency-слой
- Multi-client workspace, client health score, upsell-pipeline, share-links,
  permission-модель.

### J. Доверие / прозрачность
- Data deletion/export, audit log, дисклеймер «AI explanations generated from
  your connected data», processor-позиция (не data broker).

## ВОПРОСЫ — ПО КАЖДОЙ ФУНКЦИИ (или кластеру)

Для каждой функции из A–J оцени:
1. **Сила спроса** по каждому ICP-сегменту (high/med/low) — с доказательством
   (опросы, форумы, отзывы на конкурентов, support-вопросы).
2. **Кто уже закрывает** (конкретный конкурент) и **насколько хорошо** —
   и есть ли **дыра** (никто не делает для нашего ICP / делает плохо/дорого/сложно).
3. **Must-have / nice-to-have / differentiator** для ICP.
4. **Готовность платить**: это причина купить, причина остаться (retention),
   или причина докупить (upsell)? Ценовой сигнал, если есть.
5. **Сложность/риск** реализации с точки зрения рынка (данные дорогие? споры по
   корректности? высокая support-нагрузка?).

Особое внимание:
- **E (AI-объяснения)** — реально ли это дифференциатор, или «обёртка над GPT»,
  которую скопируют? Есть ли доказанная ценность «объяснения» (retention/ARPU)?
- **D (SEO)** — нужна ли отдельная SEO-функция нашему ICP, если есть Ahrefs/
  Semrush, или ценность только в «объяснении + связке с ROI»?
- **G+H+I (upsell/отчёты/агентства)** — это драйвер ARPU и agency-канала?

## ФОРМАТ РЕЗУЛЬТАТА

1. **Матрица «функция × ICP»**: спрос (H/M/L) + покрытие конкурентами + WTP-сигнал
   + приоритет.
2. **MoSCoW для MVP**: Must / Should / Could / Won't — с обоснованием.
3. **Killer-функция**: 1–2 функции, ради которых реально покупают (гипотеза +
   доказательство).
4. **Дыры рынка**: функции, которые никто не делает хорошо для нашего ICP =
   наш клин.
5. **Kill-list**: функции, которые НЕ строить (нет спроса / занято / дорого).
6. **Упаковка по планам**: какие функции в Starter / Growth / Agency / Enterprise.
7. **Источники** (с датами).
