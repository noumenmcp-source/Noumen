# Юр-слой США (карта, НЕ юр-консультация)

⚠️ Черновая карта требований. Перед запуском — ревью US-юристом по приватности.
Только USA. Никакого 152-ФЗ/РКН — это RF-система (см. [../SEGMENTATION.md](../SEGMENTATION.md)).

## Ключевые режимы

| Закон | Что регулирует | Влияние на экосистему |
|---|---|---|
| **CCPA / CPRA** (Калифорния) | права потребителей, sensitive PI; с 01.01.2023 покрывает **B2B-контакты** (sunset исключения) | согласие/opt-out, «Do Not Sell/Share», обработка sensitive PI (в т.ч. сигналы дохода) |
| **CAN-SPAM** | коммерческий email | физ. адрес отправителя, рабочий unsubscribe ≤10 дней, честные subject/from |
| **TCPA** | SMS/звонки/мессенджеры | prior express consent на маркетинговые сообщения; критично для модуля автоматизации мессенджеров |
| **GDPR** | если есть EU-трафик/субъекты | правовое основание, DPA, права субъектов; учесть, если US-сайт ловит EU-визитёров |
| Штатные законы | VA/CO/CT/UT и др. растущий список | универсальный consent/preference-центр вместо «по штату» |

## Что это значит для модулей

- **core-cdp:** consent-gating сбора; запись согласия; preference-центр; honoring opt-out/DSAR.
- **consent (модуль):** US-CMP — баннер/preference-центр, «Do Not Sell or Share», GPC-сигнал,
  неизменяемый журнал-доказательство (закон-агностичная ledger-механика переносится из RF-движка,
  но контент/документы — US). Это **фаза-1-зависимость**, а не поздний апселл.
- **email:** CAN-SPAM (unsubscribe, физ-адрес), double-opt-in для качества базы.
- **automation (мессенджеры):** TCPA prior express consent — без него маркетинг-сообщения нельзя.

## B2B-нюанс
CPRA с 2023 убрал B2B-исключение → деловые контакты = персональные данные. Сбор «дохода»/
чувствительных сигналов на физлиц-сотрудников = sensitive PI с доп. требованиями.
⇒ Consent-слой = условие легальности сбора данными всеми остальными модулями.
